- [ ] **Step 3: Create the backup script**

Write the following to `/opt/odoo12/backup.sh`:

```bash
#!/bin/bash
# Odoo 12 daily backup script
# Backs up Loan_Smart database and filestore with 7-day retention
# Run as odoo12 user via cron: 0 2 * * * /opt/odoo12/backup.sh

set -euo pipefail

# Configuration
DB_NAME="Loan_Smart"
FILESTORE="/opt/odoo12/.local/share/Odoo/filestore/Loan_Smart"
BACKUP_DIR="/opt/odoo12/backups"
LOG_FILE="/var/log/odoo12-backup.log"
RETENTION_DAYS=7
MIN_FREE_GB=100

# Derived
TODAY=$(date +%Y-%m-%d)
TIMESTAMP=$(date "+%Y-%m-%d %H:%M:%S")
TARGET_DIR="${BACKUP_DIR}/${TODAY}"

log() {
    echo "[${TIMESTAMP}] $1" >> "${LOG_FILE}"
}

log "=== Backup started ==="

# Disk space check (free space in GB on /opt partition)
FREE_GB=$(df --output=avail /opt | tail -1 | awk '{printf "%.0f", $1/1048576}')
if [ "${FREE_GB}" -lt "${MIN_FREE_GB}" ]; then
    log "WARNING: Only ${FREE_GB}GB free (minimum: ${MIN_FREE_GB}GB). Skipping backup."
    exit 1
fi
log "Disk check passed: ${FREE_GB}GB free"

# Create target directory
mkdir -p "${TARGET_DIR}"

# Database dump
log "Starting pg_dump..."
pg_dump "${DB_NAME}" | gzip > "${TARGET_DIR}/${DB_NAME}.sql.gz"
DB_SIZE=$(du -sh "${TARGET_DIR}/${DB_NAME}.sql.gz" | cut -f1)
log "Database dump complete: ${DB_SIZE}"

# Filestore archive
log "Starting filestore archive..."
tar czf "${TARGET_DIR}/filestore.tar.gz" -C "$(dirname "${FILESTORE}")" "$(basename "${FILESTORE}")"
FS_SIZE=$(du -sh "${TARGET_DIR}/filestore.tar.gz" | cut -f1)
log "Filestore archive complete: ${FS_SIZE}"

# Rotation — delete backups older than retention period
log "Rotating backups older than ${RETENTION_DAYS} days..."
find "${BACKUP_DIR}" -maxdepth 1 -type d -name "20*" -mtime +${RETENTION_DAYS} -exec rm -rf {} \;
REMAINING=$(ls -d "${BACKUP_DIR}"/20* 2>/dev/null | wc -l)
log "Rotation complete. ${REMAINING} backup(s) retained."

log "=== Backup finished successfully ==="
```

Use `sudo tee /opt/odoo12/backup.sh` to write the file on the server.

### Task 2: Set Up Cron Job

**Goal:** Install a cron job in the `odoo12` user's crontab to run the backup script daily at 2:00 AM UTC.

**Acceptance Criteria:**
- [ ] `odoo12` crontab contains `0 2 * * * /opt/odoo12/backup.sh`
- [ ] Cron entry is correctly formatted

**Verify:** `sudo crontab -u odoo12 -l` → shows `0 2 * * * /opt/odoo12/backup.sh`

**Steps:**

- [ ] **Step 1: Install the cron job**

```bash
(sudo crontab -u odoo12 -l 2>/dev/null; echo "0 2 * * * /opt/odoo12/backup.sh") | sudo crontab -u odoo12 -
```

This appends the new entry to any existing crontab entries for `odoo12`.

- [ ] **Step 2: Verify the cron entry**

```bash
sudo crontab -u odoo12 -l
```

Expected: `0 2 * * * /opt/odoo12/backup.sh`

- [ ] **Step 3: Verify cron service is running**

```bash
systemctl is-active cron
```

Expected: `active`
